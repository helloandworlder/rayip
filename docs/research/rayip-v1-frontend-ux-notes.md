# RayIP V1 前台 / 后台 UX 研究笔记

> 日期：2026-04-29  
> 范围：用户面板、管理面板、实时节点面板

## 1. 总原则

RayIP V1 是静态家宽代理商业平台。界面应该让客户和管理员都觉得“顺手、直接、可信”。

客户界面回答：

- 我可以买什么？
- 多少钱？
- 我有哪些代理？
- 如何复制、使用、续费、刷新？
- 余额和流量还剩多少？

管理界面回答：

- 什么可售？
- 什么在跑？
- 什么有风险？
- 什么刚发生变化？
- 哪里需要人工介入？

## 2. 用户面板

必须包含：

- 登录 / 注册
- 概览
- 充值
- 静态家宽购买
- 我的静态代理
- 代理详情
- 账单流水
- 多个开发者 App 管理：`AppID + AppSecret`
- IP 白名单 / 回调地址
- 优惠券
- 邀请返利
- 工单 / 反馈

购买页是核心页面，不是营销落地页。

## 3. 我的静态代理

表格需要支持：

- 复制 IP:Port
- 复制账号密码
- 展示协议
- 展示国家 / 城市 / 线路
- 展示 IP 类型
- 展示剩余天数
- 展示已用流量
- 续费
- 批量续费
- 凭据刷新
- IP 刷新
- 导出

客户状态保持简单：

- 待发货
- 使用中
- 续费中
- 已过期
- 已禁用
- 已退款

## 4. 管理面板

必须包含：

- 概览
- 用户
- 钱包和充值
- 订单
- 产品和价格
- 国家 / 城市 / 线路
- 库存
- 节点
- 实时节点面板
- Web SSH
- 优惠券
- 邀请规则
- 工单
- 审计日志
- 设置

管理端先展示商业状态，再展示技术细节。

## 5. 实时节点面板

参考现有节点图的体验，做成管理员专用实时面板。

展示：

- 节点组 / 线路
- 在线状态
- v4/v6 地区
- 出口 IP
- 实时上下行速度
- 开机时长
- 累计流量
- CPU / RAM / 存储
- 活跃连接数
- 最近启动、最近在线、最后同步

交互：

- 实时模式 / 历史模式切换
- 5 分钟、1 小时、24 小时、7 天趋势
- 节点详情
- Web SSH
- 禁止售卖
- 退役
- 重启 Runtime
- 触发对账

实现：

- 管理端通过 WebSocket 连 API。
- API 聚合 NodeAgent gRPC 上报。
- Redis 保存实时窗口。
- Postgres 保存聚合历史。

## 6. 设计方向

用户面板：

- 快速购买
- 强复制/导出能力
- 少运营术语
- 少解释，多动作

管理面板：

- 信息密度更高
- 表格、筛选、批量操作强
- 状态色清晰
- 操作必须审计
- 终端只在 Web SSH 页面/弹窗出现

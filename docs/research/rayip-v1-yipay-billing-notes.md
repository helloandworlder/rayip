# RayIP V1 易支付与钱包结算研究笔记

> 日期：2026-04-29  
> 范围：充值、钱包流水、下单冻结/确认/释放

## 1. 核心结论

RayIP V1 使用预付费钱包。

易支付只负责“充值入账”，不直接驱动代理发货。购买、续费、IP 刷新、凭据刷新等商业动作统一从 RayIP 钱包余额扣款。

这样可以把支付回调和代理交付解耦：

- 易支付成功后只增加钱包余额。
- 购买时先冻结余额。
- Runtime 确认代理可用后再确认扣款。
- 发货失败释放冻结余额。

## 2. 充值流程

```text
用户选择充值金额
  -> RayIP 创建充值订单
  -> RayIP 返回易支付支付链接或表单
  -> 用户完成支付
  -> 易支付回调 notify_url
  -> RayIP 校验签名、商户、金额、订单号
  -> RayIP 标记充值成功
  -> RayIP 写钱包流水并增加余额
```

`return_url` 只用于页面体验，不能作为入账依据。真正入账依据必须是异步回调。

## 3. 状态机

充值订单：

```text
CREATED -> PAYING -> PAID
CREATED -> CLOSED
PAYING  -> CLOSED
PAYING  -> FAILED
```

钱包冻结：

```text
NONE -> HELD -> CAPTURED
NONE -> HELD -> RELEASED
```

规则：

- `PAID` 必须和钱包流水在同一个数据库事务内完成。
- 同一支付回调重复到达时只能入账一次。
- 金额不一致、签名错误、订单不匹配，不能改钱包。

## 4. 钱包流水

建议流水类型：

- `RECHARGE`：充值入账
- `PURCHASE_HOLD`：购买冻结
- `PURCHASE_CAPTURE`：购买确认扣款
- `PURCHASE_RELEASE`：购买失败释放
- `RENEW_HOLD`：续费冻结
- `RENEW_CAPTURE`：续费确认扣款
- `RENEW_RELEASE`：续费失败释放
- `REFUND`：退款
- `ADJUSTMENT`：管理员调整

流水不可变。余额必须能从流水审计。

## 5. 风险控制

V1 必须有：

- 最小/最大充值金额。
- 充值创建频率限制。
- 回调签名校验。
- 金额精确匹配。
- 商户号匹配。
- 订单号唯一。
- 支付平台交易号唯一。
- 幂等键。
- 管理员手动调整审计。

## 6. 管理端

管理端展示：

- 充值订单
- 支付平台交易号
- 回调状态
- 最后回调时间
- 钱包流水
- 人工加扣款
- 异常回调列表

正常在线充值不需要人工审核。
